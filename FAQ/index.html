FAQ

Why does the job fail in your input formats?

Take a look at your data. Make sure there are no \t's anywhere or any other characters that might cause the input to fail.

How do I format my data, so it will work with DGA?

Using your own data with DGA is simple! Make sure the file is an edge list of the following format.

Source,Target,EdgeValue

It must be a character separated file, so you can use commas, TABS, etc. If you use anything other than the default, then you must let us know via a configuration value.

I found a bug! How can I let you know?

Bug Reports can be submitted here. We will try and get it resolved as soon as possible!
