FAQ

Why does the job fail in Giraph?

There is a good chance you got a bad commit when you pulled down giraph. This is the trade off when you use SNAPSHOT builds. We've tested it using this commit checksum: f0b6cddd3e255e9e31e3afe22d0f69b3c046d554

To switch back to that commit run the following command below.


    $ git reset --hard f0b6cddd3e255e9e31e3afe22d0f69b3c046d554

Why does the job fail in your input formats?

Take a look at your data. Make sure there are no \t's anywhere or any other characters that might cause the input to fail.

Why don't my -D parameters work?

This is a difference between the DGARunner and GiraphRunner. In Giraph, you used -Dgiraph.zkList=localhost:2181. In DGA, you do -D giraph.zkList=localhost:2181.

I found a bug! How can I let you know?

Bug Reports can be submitted here. We will try and get it resolved as soon as possible!
